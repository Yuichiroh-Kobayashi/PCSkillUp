# ドット絵PC操作ミッション

## 1. 教材名
ドット絵PC操作ミッション

## 2. 目的
- 中学校技術科（情報分野）の最初のPC室授業で、マウス操作（クリック・ドラッグ）、ファイル保存、手動の Ctrl+C / Ctrl+V を短時間で練習する。
- ドットの色数やマス数の違いを通して、情報量とデータ量の関係へ接続する。

## 3. モード
- 練習モード: 16×16、2色。見本の「1」があるマスを黒く塗る。
- 上級モード: 32×32、4色。パレットで「白色・銀色・灰色・黒色」を選び、見本に合わせて塗る。

## 4. 使い方
1. 見本を見ながら、作業グリッドをクリックまたはドラッグで塗る。
2. 上級モードでは、先にパレットで色を選んでからマスを塗る。
3. 必要に応じて「全部消す」で最初からやり直す。
4. 「今の絵をBMP保存」で作業グリッドをBMPファイルとして保存する。
5. 下の文をドラッグで選択し、Ctrl+C でコピーする。
6. 貼り付け欄に Ctrl+V で貼り付ける。
7. 「ドット絵完成」＋「指定文の貼り付け成功」でミッション完了メッセージが表示される。

手動の Ctrl+C / Ctrl+V 練習は、この教材のコア活動です。自動コピーボタンや0/1データの可視テキストエリアは使いません。

## 5. BMP保存
BMP保存は、練習モードと上級モードの両方で使えます。

目的は次の2つです。

1. 保存されたファイルやファイル名を扱う練習をする。
2. マス数や色数が増えると、扱う情報量・データ量も増えることを体験する。

練習モードは16×16のモノクロBMP、上級モードは32×32の16色BMPとして保存します。上級モードの教材上の色は4色ですが、Windows Paintとの互換性を重視し、4bppのインデックスBMPとして16個のパレット枠を持たせています。

ブラウザや学校PCの設定によっては、「名前を付けて保存」ダイアログが出ず、自動的にダウンロードフォルダへ保存される場合があります。アプリ側でファイル名は指定しますが、実際の保存動作はブラウザと端末設定に依存します。

## 6. Undoについて
授業試行では、生徒が同じマスを何度もクリックして調整する行動が多く、「1つ戻す」は復帰手段として有効に働きませんでした。そのため、Undo / 「1つ戻す」は意図的に提供していません。

## 8. ローカルでの確認方法
Python が使える環境で以下を実行します。

```bash
cd docs
python3 -m http.server 8000
```

ブラウザで `http://localhost:8000` を開いて動作確認します。

## 9. 授業での見取り観点
- 見本を見て、クリック/ドラッグで塗り分けできるか。
- 上級モードで、パレットを先に選んでから塗れるか。
- 保存したBMPファイルとファイル名を確認できるか。
- 文をドラッグ選択して Ctrl+C できるか。
- 貼り付け欄へ Ctrl+V できるか。
- 色数やマス数の違いとデータ量の関係に気づけるか。

## 10. 個人情報・外部通信なしの方針
- ログイン不要。
- このアプリ自体は、個人情報の入力・保存・送信を行いません。
- 外部API、解析タグ、トラッカー、CDN、外部画像、外部フォントは使用していません。
- ただし、GitHub Pagesで公開する場合、通常のWebアクセスとしてGitHub側のアクセスログは発生します。
- HTML/CSS/JavaScript の静的ファイルのみで動作します。

## 11. 既知の制約
- 任意の文字・絵文字の変換はMVPでは未対応。
- タイピング練習はPlaygram Typingを使う想定。
- ネットワークコマンド練習は対象外。
