# ドット絵PC操作ミッション

## 1. 教材名
ドット絵PC操作ミッション

## 2. 目的
- 中学校技術科（情報分野）の最初のPC室授業で、**マウス操作（クリック・ドラッグ）**と**Ctrl+C / Ctrl+V**の基礎を短時間で練習する。
- 黒白のドット絵を 0/1 データに対応づけることで、次時の「情報のデジタル化（2進数表現）」へ接続する。

## 3. 対象授業
- 中学校 技術・家庭科（技術分野）
- 情報の基礎
- PC室初回（5〜8分程度の導入ミッション）

## 4. 使い方
1. 見本（16x16）を見て、作業グリッド（16x16）のマスをクリックして白黒を切り替える。
2. ドラッグで連続して塗る。
3. 必要に応じて「1つ戻す」「全部消す」でやり直す。
4. 完成したら「0/1データをコピー」を押す。
5. 下の貼り付け欄に **Ctrl+V** で貼り付ける。
6. 見本一致＋貼り付け一致でミッション完了メッセージが表示される。

## 5. GitHub Pagesでの公開想定
- ブランチ: `main`
- 公開フォルダ: `/docs`
- アプリ本体: `docs/index.html`

## 6. ローカルでの確認方法
Python が使える環境で以下を実行します。

```bash
cd docs
python3 -m http.server 8000
```

ブラウザで `http://localhost:8000` を開いて動作確認します。

## 7. 授業での見取り観点
- クリックで白黒を切り替えられるか。
- ドラッグで連続操作できるか。
- 間違えたときに「1つ戻す」「全部消す」で復帰できるか。
- Ctrl+C / Ctrl+V の一連の流れを実行できるか。
- 黒=1、白=0 の意味を口頭説明できるか（次時接続）。

> 教師向けメモ: 速度競争ではなく、操作の安定と復帰行動（やり直し）を重視する導入教材です。

## 8. 個人情報・外部通信なしの方針
- ログイン不要。
- 個人情報の入力・保存・送信なし。
- 外部API / 解析タグ / トラッカー / CDN / 外部画像 / 外部フォント不使用。
- HTML/CSS/JavaScript の静的ファイルのみで動作。

## 9. 既知の制約
- 任意の文字・絵文字の変換はMVPでは未対応。
- タイピング練習はPlaygram Typingを使う想定。
- ネットワークコマンド練習は対象外。

## 10. 今後の拡張案
- 任意文字を16x16ドット絵に変換。
- 絵文字を白黒ドット絵に変換。
- 8x8 / 16x16 / 32x32 の情報量比較。
- 授業用ワークシートとの連携。
- 教師用の観察チェックリスト。
