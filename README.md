# ドット絵PC操作ミッション

## 1. 教材名
ドット絵PC操作ミッション

## 2. 目的
- 中学校技術科（情報分野）の最初のPC室授業で、**マウス操作（クリック・ドラッグ）**と**Ctrl+C / Ctrl+V**の基礎を短時間で練習する。
- 「1」の場所を黒く塗る活動を通して、白黒の組み合わせで絵を表せることを体験し、次時の「情報のデジタル化」へ接続する。

## 3. 対象授業
- 中学校 技術・家庭科（技術分野）
- 情報の基礎
- PC室初回（5〜8分程度の導入ミッション）

## 4. 使い方
1. 見本（16x16）を見ながら、作業グリッドの「1」がある場所をクリックまたはドラッグで黒く塗る。
2. 必要に応じて「1つ戻す」「全部消す」でやり直す。
3. 下の文をドラッグで選択し、**Ctrl+C** でコピーする。
4. 貼り付け欄に **Ctrl+V** で貼り付ける。
5. 「ドット絵完成」＋「指定文の貼り付け成功」でミッション完了メッセージが表示される。

> この版では、0/1データ欄は使いません。コピー練習は自動コピーではなく、手動の選択＋Ctrl+Cで行います。

## 6. ローカルでの確認方法
Python が使える環境で以下を実行します。

```bash
cd docs
python3 -m http.server 8000
```

ブラウザで `http://localhost:8000` を開いて動作確認します。

## 7. 授業での見取り観点
- 「1」の場所を見て、クリック/ドラッグで塗り分けできるか。
- 間違えたときに「1つ戻す」「全部消す」で復帰できるか。
- 文をドラッグ選択して Ctrl+C できるか。
- 貼り付け欄へ Ctrl+V できるか。
- 白黒の組み合わせで絵を表せることを説明できるか。

> 教師向けメモ: 速度競争ではなく、操作の安定と復帰行動（やり直し）を重視する導入教材です。

## 8. 個人情報・外部通信なしの方針
- ログイン不要。
- このアプリ自体は、個人情報の入力・保存・送信を行いません。
- 外部API、解析タグ、トラッカー、CDN、外部画像、外部フォントは使用していません。
- ただし、GitHub Pagesで公開する場合、通常のWebアクセスとしてGitHub側のアクセスログは発生します。
- HTML/CSS/JavaScript の静的ファイルのみで動作。

## 9. 既知の制約
- 任意の文字・絵文字の変換はMVPでは未対応。
- タイピング練習はPlaygram Typingを使う想定。
- ネットワークコマンド練習は対象外。

## 10. 今後の拡張案
- 任意文字を16x16ドット絵に変換。
- 絵文字を白黒ドット絵に変換。
- 8x8 / 16x16 / 32x32 の情報量比較。
- 授業用ワークシートとの連携。
- 教師用の観察チェックリスト。
